## 🎯 Objective
The objective of this task is to perform a vulnerability assessment on a sample web application using OWASP ZAP, identify security misconfigurations, and classify the risks based on severity levels (High, Medium, Low). The findings are documented in a structured report for security analysis and reporting practice.

---

## 🛠 Tools Used
- OWASP ZAP (Zed Attack Proxy)
- Google Chrome Browser
- Kali Linux / Windows Environment
- Target Application: https://demo.owasp-juice.shop/

---

## 🌐 Target Application
https://demo.owasp-juice.shop/

---

## 🔍 Methodology
1. Launched OWASP ZAP tool
2. Performed automated scan on the target application
3. Crawled application endpoints and resources
4. Identified security misconfigurations and vulnerabilities
5. Analyzed alerts generated by OWASP ZAP
6. Classified vulnerabilities based on severity levels
7. Captured screenshots as evidence for reporting

---

## 🚨 Identified Vulnerabilities

The following security issues were identified during the scan:

- Content Security Policy (CSP) Header Not Set
- Cross-Domain Misconfiguration (CORS)
- Server Leaks Version Information via HTTP Response Headers
- Strict-Transport-Security (HSTS) Header Not Set
- Timestamp Disclosure (Unix Format)
- Modern Web Application Information Disclosure
- Cache-Control Directives Not Properly Configured

---

## 📊 Risk Classification

- 🔴 High Risk: Critical vulnerabilities that may lead to system compromise
- 🟠 Medium Risk: Issues that may be exploited under certain conditions
- 🟡 Low Risk: Informational or minor security misconfigurations
